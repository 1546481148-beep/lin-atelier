当页面已经“能用”以后，真正拉开差距的往往不是更复杂的功能，而是细节有没有被认真处理。

## 1. 点击之后有没有明确反馈

最常见的问题不是动画太少，而是用户做了动作以后，页面没有给出任何回应。

- 按钮状态有没有变化
- 是否出现加载中的中间态
- 结果出现前，是否有过渡而不是瞬间跳变

## 2. 层级变化是否能被感知

弹层、切换面板、展开模块都涉及层级变动。如果用户需要花一秒才意识到“刚刚发生了什么”，说明视觉节奏还不够清楚。

## 3. 动效是否在帮助阅读

我会过滤掉那些只是为了“看起来高级”的动效，只保留真的在帮助用户理解结构变化的部分。

```js
const intent = "clarity-first";
const animation = intent === "clarity-first" ? "keep" : "remove";
```

## 4. 结束状态是否干净

很多交互开始得不错，但结束得含糊。动画收尾、焦点回归、按钮状态复原，这些都决定了体验是不是完整。

这份清单不会替你设计页面，但会让你每次上线前都多一次“把手感打磨完”的机会。
